# 哈希缓存全链路集成实施计划

## 摘要

为 fuscan 实现基于内容哈希的扫描结果缓存：规则按条目哈希、文件按内容哈希，缓存键为 `(file_hash, rule_hash)`，路径无关。cache 模块（schema/hashes/store）已实现并通过 56 个测试（99.62% 覆盖率），本计划覆盖剩余的 Scanner/ArchiveScanner/IncrementalScanner/CLI/GUI/Config 全链路集成与文档。

- 当 `cache is not None`：
  - 调 `cache.register_ruleset(ruleset, source_files)` 登记规则
  - 存 `self._cache = cache`
  - 存 `self._rule_hashes = cache.get_rule_hashes()`（或 register_ruleset 返回值）
  - 构造 `self._compiled_with_hash: list[tuple[Rule, Matcher, str]] = [(rule, matcher, self._rule_hashes[rule.name]) for rule, matcher in self._compiled]`

  - 构造 ArchiveScanner 时传入 `cache=cache`
- 当 `cache is None`：维持原行为
- `_scan_entry` 改为分流：
  ```python
  def _scan_entry(self, entry: FileEntry) -> ScanResult:
      if self._cache is None:
          return self._scan_entry_uncached(entry)
      return self._scan_entry_cached(entry)
  ```
- 原 `_scan_entry` 逻辑改名为 `_scan_entry_uncached`
- 新增 `_scan_entry_cached`：
  1. 调 `self._hashing_content_provider(entry)` 取 `(content, file_hash)`——一次 I/O
  2. 构造静态 `content_provider` 闭包返回 `content`，传给 `MatchContext`（不改 MatchContext）
  3. 扩展名过滤后收集 `applicable = [(rule, matcher, rule_hash), ...]`
  4. 批量 `cache.get_cached_hits(file_hash, [rh for _,_,rh in applicable])`
  5. 遍历 applicable：命中缓存的直接构造 RuleHit（`rule_name=rule.name` 填回）；未命中走 `matcher.matches`，命中调 `put_result(file_hash, rule_hash, hit)`，未命中调 `put_result(file_hash, rule_hash, None)`
  6. 调 `register_file(file_hash, entry.size)` + `register_path(file_hash, entry.path, entry.mtime)`

**scanner/__init__.py**：导出 `default_extract_content_with_hash`

## 假设与决策

1. **缓存键粒度**：`(file_hash, rule_hash)` 单条级，非整文件级。同一文件对部分规则命中、部分未命中可混合处理。
2. **路径无关**：`scanned_files` 以 `file_hash` 为主键，`file_paths` 登记多个路径。文件移动后哈希不变 → 缓存仍命中。
3. **未命中也缓存**：`put_result(file_hash, rule_hash, None)` 记录"该规则对该文件未命中"，避免重复扫描。
4. **ArchiveScanner 不独立注册规则**：由父 Scanner 调 `register_ruleset`，ArchiveScanner 调 `get_rule_hashes()` 复用。
5. **IncrementalScanner 接口保留**：`save_state`/`load_state` 改空操作，避免破坏 tray.py；缓存 DB 是真正的持久化。
6. **builtin 规则来源**：`source_files` 仅含用户规则文件路径；builtin 规则归入 `__inline__` 虚拟来源（store 已处理）。
7. **并发安全**：CacheStore 内部 RLock 串行化所有读写，Scanner 多线程调用安全。
8. **HashingContentProvider 不改 MatchContext**：缓存模式下先调 hashing provider 取 `(content, hash)`，再用静态闭包包装 content 传给 MatchContext。
