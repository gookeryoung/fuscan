# 哈希缓存全链路集成 - 实施计划

## 摘要

为 fuscan 实现基于内容哈希的扫描结果缓存，建立"规则文件（文件名、哈希）↔ 被扫描文件（路径、文件哈希）↔ 扫描结果"的持久化关联。核心机制：每次扫描算文件 SHA-256 和规则哈希，缓存键为 `(file_hash, rule_hash)`；下次扫描时先查缓存，文件哈希和规则哈希都未变则直接复用结果，跳过匹配器。路径无关，规则变更感知。

本计划覆盖**剩余集成工作**（任务 8-12）。缓存子包（`fuscan.cache`）、Scanner 缓存集成（`scanner.py`）、ArchiveScanner 缓存集成（`archive/scanner.py`）及对应测试**已在前期完成**，本计划从验证覆盖率起步，然后接入 IncrementalScanner / CLI / GUI / Config / 文档。

## 变更详情

### 任务 8：IncrementalScanner 替换为哈希缓存委托

**文件**：`src/fuscan/watcher/incremental.py`

**当前问题**：IncrementalScanner 独立实现 mtime 跟踪 + 自有 `_scan_entry`，不使用 Scanner，也不接入缓存。

**变更内容**：

1. **构造函数**：加 `cache: CacheStore | None = None` 参数；构造 `Scanner` 实例（传 `cache` 和 `source_files`）；删除 `_compiled`/`_has_unrestricted_rule`/`_all_extensions`/`_file_states`（委托 Scanner 后不再需要）。

2. **`scan(root)`**：直接 `return self._scanner.scan(root)`。

3. **`scan_paths(paths)`**：遍历 paths，对每个存在的文件调 `self._scanner.scan_file(path)`，聚合为 `ScanReport`。

4. **`tracked_count` 属性**：`cache` 不为 None 时返回 `cache.stats().scanned_files`，否则返回 0。

5. **`mark_scanned`/`remove_path`/`save_state`/`load_state`**：变为空操作（`pass`），保留签名兼容 TrayApp 调用。`cache` 持久化由 SQLite 处理。

6. **`_should_scan`/`_scan_entry`**：删除（委托 Scanner）。

**关键代码结构**：

```python
class IncrementalScanner:
    def __init__(self, ruleset, ..., cache: CacheStore | None = None):
        self._cache = cache
        self._scanner = Scanner(
            ruleset, cache=cache, source_files=...,  # 由调用方传 source_files
            max_depth=max_depth, scan_archives=scan_archives,
            ignore_dirs=ignore_dirs, ignore_extensions=ignore_extensions,
        )

    @property
    def tracked_count(self) -> int:
        return self._cache.stats().scanned_files if self._cache else 0

    def scan(self, root: Path) -> ScanReport:
        return self._scanner.scan(root)

    def scan_paths(self, paths: list[Path]) -> ScanReport:
        # 遍历 paths，调 scan_file，聚合结果
        ...

    def mark_scanned(self, path, mtime) -> None: pass
    def remove_path(self, path) -> None: pass
    def save_state(self, path) -> None: pass
    def load_state(self, path) -> None: pass
```

**注意**：`source_files` 需由 IncrementalScanner 的调用方（TrayApp）传入。为简化接口，在 `__init__` 中自动计算：若 `cache` 不为 None，根据 `ruleset` 和已知规则文件路径计算 `source_files`。但 IncrementalScanner 不知道规则文件路径，所以**改由 TrayApp/CLI/GUI 计算 source_files 后通过 Scanner 直接传递**。

**简化方案**：IncrementalScanner 不传 `source_files`（传 None），Scanner 内部 `register_ruleset` 会用 `__inline__` 虚拟来源登记规则。规则哈希仍正确计算（基于规则内容，不依赖文件路径）。`source_files` 的价值仅在"规则文件变更时跟踪来源"，对缓存命中无影响。因此 IncrementalScanner 传 `source_files=None` 可接受。

   ```python
   def _compute_source_files(rules_paths: list[Path], use_builtin: bool) -> dict[Path, str]:
       """计算规则文件 SHA-256 映射。"""
       from fuscan.cache import hash_bytes
       sources: dict[Path, str] = {}
       if use_builtin:
           from fuscan.builtin import BUILTIN_RULES_PATH
           if BUILTIN_RULES_PATH.exists():
               sources[BUILTIN_RULES_PATH] = hash_bytes(BUILTIN_RULES_PATH.read_bytes())
       for p in rules_paths:
           if p.exists():
               sources[p] = hash_bytes(p.read_bytes())
       return sources
   ```
