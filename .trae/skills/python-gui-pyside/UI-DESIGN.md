# PySide GUI UI 设计规范

基于 SKILL.md 设计令牌的 UI 设计规范。令牌定义见 SKILL.md「设计系统」，本文档定义令牌的使用场景、组件设计、交互模式与状态规范。

## 设计原则

1. **简洁优先**：每个界面只承载单一核心任务，次要操作收入次级菜单或折叠面板；避免功能堆砌。
2. **功能整合**：相关功能聚合到同一面板，用分组标题分隔而非分散多页；一个操作流程在一个界面内完成。
3. **关联展示**：按当前任务阶段只显示相关控件，无关控件隐藏或禁用；上下文变化时动态调整可见区域。
4. **渐进披露**：高级选项默认折叠，常用操作优先暴露；详情按需展开，不一次呈现所有信息。

## 色彩使用

### 配色比例

遵循 60-30-10 原则：
- **60% 中性底色**：`COLOR_BG_APP`（内容区）+ `COLOR_BG_MUTED`（分组间隙）
- **30% 主色**：`COLOR_PRIMARY`（头部条、侧边栏、主操作）+ `COLOR_TEXT_PRIMARY`（正文）
- **10% 强调**：`COLOR_ACCENT`（选中态高亮）+ 语义色（状态提示）

### 语义色使用场景

| 语义 | 令牌 | 使用场景 |
|------|------|----------|
| 主操作 | `COLOR_PRIMARY` | 主按钮、头部条、侧边栏、Tab 选中下划线 |
| 成功 | `COLOR_SUCCESS` / `COLOR_ACCENT` | 操作完成提示、成功状态图标 |
| 警告 | `COLOR_WARNING` | 非破坏性提示、待确认状态 |
| 错误 | `COLOR_DANGER` | 错误消息、危险操作按钮、校验失败边框 |
| 禁用 | `COLOR_TEXT_SECONDARY` + `COLOR_BG_MUTED` | 禁用控件文字与背景 |

### 状态色映射

| 控件状态 | 背景色 | 文字色 |
|----------|--------|--------|
| 默认 | `COLOR_BG_APP` | `COLOR_TEXT_PRIMARY` |
| 悬停 | `COLOR_BG_MUTED` | `COLOR_TEXT_PRIMARY` |
| 按下 | `COLOR_PRIMARY_DARK` | `COLOR_TEXT_ON_PRIMARY` |
| 选中 | `COLOR_PRIMARY` | `COLOR_TEXT_ON_PRIMARY` |
| 禁用 | `COLOR_BG_MUTED` | `COLOR_TEXT_SECONDARY` |
| 错误 | `COLOR_BG_APP` + `COLOR_DANGER` 边框 | `COLOR_DANGER` |

## 排版使用

### 字号层级

| 层级 | 令牌 | 使用场景 | 规则 |
|------|------|----------|------|
| L1 标题 | `FONT_TITLE` | 窗口标题、页面标题 | 每页仅 1 个 |
| L2 标题 | `FONT_HEADING` | 区块标题、分组标题、对话框标题 | 每区块 1 个 |
| L3 正文 | `FONT_BODY` | 正文、表单标签、按钮、列表项 | 默认字号 |
| L4 说明 | `FONT_CAPTION` | 状态栏、表头、辅助说明、时间戳 | 不用于关键信息 |

### 行高与对齐

- 正文行高 1.5 倍字号（`FONT_BODY` 约 20px）；标题行高 1.3 倍。
- 表单标签右对齐，输入内容左对齐；数值右对齐，文本左对齐。
- 长文本截断用省略号 `…`，悬停显示完整内容（`setToolTip`）。

## 间距与布局

### 8px 网格应用

所有间距须为 8 的倍数，使用 `SPACING_*` 令牌：

| 场景 | 令牌 | 示例 |
|------|------|------|
| 图标与文字间隙 | `SPACING_XS` (4px) | 按钮内图标+文字 |
| 控件内边距 | `SPACING_SM` (8px) | 输入框内边距、按钮内边距 |
| 控件间间隙 | `SPACING_MD` (16px) | 表单字段间距、按钮组间距 |
| 区块内边距 | `SPACING_LG` (24px) | 卡片内边距、面板内边距 |
| 区块间间隙 | `SPACING_XL` (32px) | 卡片之间、大区块之间 |

### 表单布局

- 标签在上方（`QFormLayout`，`LabelRole` 顶部对齐），不用左侧标签（宽度不一致影响扫视）。
- 字段间距 `SPACING_MD`；分组间距 `SPACING_XL`。
- 必填项标签后加 `*`（`COLOR_DANGER` 色）。
- 输入框高度统一 `CONTROL_HEIGHT`，紧凑模式 `CONTROL_HEIGHT_SM`。

### 卡片布局

- 卡片背景 `COLOR_BG_APP`，圆角 `RADIUS_MD`，边框 1px `COLOR_BORDER`。
- 卡片内边距 `SPACING_LG`；卡片间距 `SPACING_XL`。
- 卡片标题用 `FONT_HEADING`，标题与内容间距 `SPACING_MD`。
- 内容区用 `QGridLayout`，`resizeEvent` 中重算列数（宽 ≥ 1200px 三列，≥ 800px 两列，否则一列）。

## 组件设计

### 按钮

| 类型 | 背景 | 文字 | 用途 |
|------|------|------|------|
| 主按钮 | `COLOR_PRIMARY` | `COLOR_TEXT_ON_PRIMARY` | 每个面板仅 1 个主操作（确认/提交） |
| 次按钮 | `COLOR_BG_APP` + 边框 | `COLOR_TEXT_PRIMARY` | 取消、次要操作 |
| 危险按钮 | `COLOR_DANGER` | `COLOR_TEXT_ON_PRIMARY` | 删除、清空等不可逆操作 |
| 文本按钮 | 透明 | `COLOR_PRIMARY` | 内联轻量操作（查看详情、展开） |

- 高度 `CONTROL_HEIGHT`，紧凑 `CONTROL_HEIGHT_SM`；圆角 `RADIUS_SM`。
- 按钮组右对齐，主按钮在最右；水平间距 `SPACING_SM`。
- 带图标的按钮，图标在左文字在右，间隙 `SPACING_XS`。

### 输入框

- 高度 `CONTROL_HEIGHT`，圆角 `RADIUS_SM`，边框 1px `COLOR_BORDER`。
- 聚焦时边框变 `COLOR_PRIMARY`（2px）；校验失败边框 `COLOR_DANGER`。
- 占位符用 `COLOR_TEXT_SECONDARY` + `FONT_CAPTION`。
- 长输入框下方显示字数计数（`FONT_CAPTION`，右对齐）。

### 对话框

- 模态对话框用于：确认破坏性操作、阻塞式输入（需立即响应）。
- 非模态对话框用于：查找替换、属性面板、可并行操作的辅助窗口。
- 按钮布局：主操作在右，取消在左；危险操作前加确认复选框或二次确认。
- 对话框标题用 `FONT_HEADING`，内容用 `FONT_BODY`，内边距 `SPACING_LG`。
- 最大宽度不超过主窗口 60%，最小宽度 400px。

### 列表与表格

- 行高 `CONTROL_HEIGHT`（紧凑 `CONTROL_HEIGHT_SM`）；表头高 `CONTROL_HEIGHT`。
- 选中行背景 `COLOR_PRIMARY` + 文字 `COLOR_TEXT_ON_PRIMARY`；悬停行 `COLOR_BG_MUTED`。
- 表头背景 `COLOR_BG_MUTED`，文字 `COLOR_TEXT_SECONDARY` + `FONT_CAPTION`。
- 空列表显示空状态插画 + 引导文案（见「状态设计」）。
- 表格列宽可拖拽调整，关键列固定不折叠。

### 卡片

- 信息卡片：标题 + 摘要 + 操作按钮，用于概览页。
- 设置卡片：分组标题 + 表单字段，用于设置页。
- 卡片间用 `SPACING_XL` 分隔；卡片内分区用 1px `COLOR_BORDER` 分割线。

## 交互设计

### 动画与过渡

- 过渡时长 150-300ms（短动画用于状态切换，长动画用于面板展开）。
- 缓动函数用 `QEasingCurve.OutCubic`（淡入淡出）或 `OutQuint`（滑动）。
- 侧边栏折叠/展开动画 200ms；Tab 切换 150ms 淡入。
- 禁用超过 500ms 的动画（桌面应用要求即时响应）。

### 反馈机制

| 场景 | 反馈方式 | 时长 |
|------|----------|------|
| 按钮点击 | 按下态视觉变化 | 即时 |
| 短任务（< 2s） | 按钮变 loading + 禁用 | 任务完成恢复 |
| 长任务（2-10s） | 模态进度对话框 + 取消按钮 | 可取消 |
| 超长任务（> 10s） | 后台执行 + 状态栏进度 + 通知 | 完成后通知 |
| 操作成功 | 状态栏绿色提示 2s 自动消失 | 2s |
| 操作失败 | 对话框 + 错误详情 + 重试按钮 | 需手动关闭 |

### 状态设计

| 状态 | 显示内容 | 设计要求 |
|------|----------|----------|
| 空状态 | 插画 + 标题 + 引导操作 | 居中显示，插画 64×64，引导按钮为次按钮 |
| 加载中 | Spinner + 说明文字 | 居中，说明用 `FONT_CAPTION` |
| 错误状态 | 图标 + 错误信息 + 重试按钮 | `COLOR_DANGER` 图标，错误信息 `FONT_BODY` |
| 无权限 | 锁图标 + 说明 | 禁用相关操作，显示原因 |
| 网络断开 | 状态栏警告条 | `COLOR_WARNING` 背景，`COLOR_TEXT_ON_PRIMARY` 文字 |

## 导航设计

### 侧边栏

- 一级导航用 `QListWidget`，每项 40px 高，图标 20×20 + `FONT_BODY`。
- 选中项左侧 3px `COLOR_ACCENT` 竖条 + 背景 `COLOR_PRIMARY_DARK`。
- 侧边栏宽度 `SIDEBAR_WIDTH`；宽 < 1000px 时折叠为图标条（56px），仅显示图标。
- 导航项按业务流程从上到下排列；不相关功能分组用分隔线。

### 面包屑

- 路径用 `>` 分隔，`FONT_CAPTION` + `COLOR_TEXT_SECONDARY`。
- 当前页 `COLOR_TEXT_PRIMARY` + Bold；上级页可点击，悬停 `COLOR_PRIMARY`。
- 超过 3 级时中间用 `…` 折叠，点击展开。

### Tab 切换

- 次级 `QTabWidget`，Tab 高 36px，`FONT_BODY`。
- 选中 Tab 底部 2px `COLOR_PRIMARY` 下划线；未选中无下划线。
- Tab 内容区边框 1px `COLOR_BORDER`，圆角 `RADIUS_MD`。
- Tab 数量超过 6 个时改为下拉选择 + 面包屑。

### 关联设计（阶段切换）

- 根据当前任务阶段动态调整侧边栏可见项（如：配置阶段只显示配置相关导航）。
- 阶段切换时清空内容区无关状态，只保留阶段上下文。
- 非当前阶段的功能入口禁用（`COLOR_TEXT_SECONDARY`）而非隐藏，保留可发现性。

## 响应式策略

### 断点定义

| 宽度范围 | 布局调整 |
|----------|----------|
| ≥ 1280px | 侧边栏展开 + 三列卡片 + 工具栏完整 |
| 1000-1279px | 侧边栏展开 + 两列卡片 + 工具栏完整 |
| 800-999px | 侧边栏折叠为图标条 + 两列卡片 + 工具栏精简 |
| < 800px | 不支持（最小尺寸 800×600） |

### 适配规则

- `resizeEvent` 中重算卡片列数与侧边栏折叠状态。
- 工具栏按钮在窄屏时收入「更多」下拉菜单。
- 表格在窄屏时隐藏次要列，保留关键列。
- 表单在窄屏时从两列布局变单列布局。

## 可访问性

### 对比度

- 正文文字对比度 ≥ 4.5:1（WCAG AA）；大号文字（≥ 18px）≥ 3:1。
- `COLOR_TEXT_SECONDARY`（`#518394`）在 `COLOR_BG_APP`（`#FFFFFF`）上对比度约 4.6:1，符合 AA。
- 禁用态文字对比度 ≥ 3:1（`COLOR_TEXT_SECONDARY` 在 `COLOR_BG_MUTED` 上）。

### 键盘导航

- 所有操作可通过键盘完成：Tab 切换焦点，Enter/Space 激活，Esc 关闭对话框。
- 焦点可见：聚焦控件显示 2px `COLOR_PRIMARY` 轮廓。
- 快捷键：Ctrl+B 折叠侧边栏，Ctrl+W 关闭 Tab，Ctrl+, 打开设置。
- 菜单项设置 mnemonic（`&文件`、`&编辑`）。

### 字体

- 最小字体 11px（`FONT_CAPTION`），不使用更小字号。
- 支持系统字体缩放（DPI 感知）：`app.setAttribute(Qt.AA_EnableHighDpiScaling)`。
- 长文本不截断关键信息，用 Tooltip 或换行显示完整内容。

## 文案规范

### 按钮文案

- 用动词 + 名词：「保存文件」「删除项目」「导出数据」。
- 主按钮 2-4 字，不超过 6 字；危险按钮加对象名「删除项目」。
- 不用「确定」「取消」以外的模糊词；不用「OK」「Yes」「No」。

### 错误消息

- 结构：[发生了什么] + [为什么] + [怎么办]。
- 示例：「无法保存文件：路径 D:\ 不存在。请选择其他路径后重试。」
- 不用技术术语暴露给用户（如堆栈、异常类名）；不指责用户。

### 空状态文案

- 结构：[当前状态] + [引导操作]。
- 示例：「还没有项目。点击「新建项目」开始。」
- 引导操作用次按钮，不用主按钮（避免误操作）。

### 确认对话框

- 危险操作必须确认：「确定删除项目「xxx」？此操作不可撤销。」
- 确认按钮用危险色 + 操作名（「删除项目」），不用「确定」。
- 可撤销操作不确认，直接执行并提供撤销入口。

## 与 SKILL.md 的关系

- SKILL.md「设计系统」定义令牌值；本文档定义令牌的使用场景与组合规则。
- SKILL.md「代码模板」提供组件实现；本文档定义组件的视觉与交互规范。
- SKILL.md「常见陷阱」列出技术禁忌；本文档列出设计禁忌。
- 开发时先查本文档确定设计规范，再查 SKILL.md 获取代码模板。
